### 1-2. 参考知見（出典はあるが検証未完了※）

- **Prodigy Math（米・RPG型算数アプリ、小学生）の実地研究**［[ERIC ED611319](https://eric.ed.gov/?id=ED611319)］
  - 利用量（回答数）と州テスト成績の伸びに有意な正の相関。**約888問の回答≒パーセンタイル+1**という用量効果
  - ただし**中利用と高利用で差がなく、一定量以上は頭打ち**（→ 無制限に遊ばせる意味はない。1日の適量設計が正しい）
  - 子供を最も惹きつけた要素：**レベルアップの達成感／モンスター討伐／友達と同じ世界にいる感覚／アバターとペットへの愛着**
  - **失敗パターン**：つまずいた子が助けを求めずゲームを続け、同じ問題の再提示と答え表示だけで**ゲーム内補習が欠如**→ 解説・類題・大人への通知が必要
- 見た目や演出だけで**ゲームメカニクス（ルール・目標・報酬）を欠く装飾的ゲーム化は学習に逆効果**になりうる［[PMC10591086](https://pmc.ncbi.nlm.nih.gov/articles/PMC10591086/)］
- ゲーミフィケーションは**外発的動機（ごほうび）に強く働き、内発的動機（面白いから学ぶ）への効果は相対的に弱い**→ 報酬漬けにせず「上達の実感」を演出する設計が必要［[Wiley 2025メタ分析](https://onlinelibrary.wiley.com/doi/10.1002/pits.70056)］
- 同メタ分析では**中等教育（中1相当）で効果大、初等教育（小4相当）では効果小**の傾向 → 小4には動機づけの仕掛け（ペット・収集・物語）を厚めにする根拠
- Classcraft等のゲーミフィケーション・プラットフォームは学習到達度に中程度の効果（d=0.621、27実験のメタ分析）［[Wiley 2021](https://onlinelibrary.wiley.com/doi/10.1155/2021/9429112)］

※検証エージェントの一部がセッション上限で実行できず「未検証」扱い。数値を意思決定に使う場合は再検証推奨（上限リセット後に再実行可能）。

### 1-3. リサーチから導く設計原則（この設計書の背骨）

1. **学習が主・ゲームが従**：バトル＝問題を解くこと。問題を解かずに進める要素を作らない（装飾的ゲーム化の逆効果を回避）
2. **拡張版設計**：不正解の直後に解説→類題で「リベンジ」までをバトル内に組み込む（Prodigyの失敗＝補習欠如への対策）
3. **毎日少しずつ**：1回10〜20分のダンジョン＋スタミナ制。長時間の周回に意味を持たせない（頭打ち効果・スクリーンタイム対応）
4. **上達の実感を演出**：レベル・討伐数・「弱点克服」の可視化で内発的動機（有能感）を育てる。報酬は見た目・収集に限定し、学習のショートカットは売らない
5. **つまずき検知**：連続不正解で自動ヒント＋敵弱体化、繰り返すなら保護者LINE通知（既存インフラ流用）
6. **偏差値65をゲームの最終目標として可視化**：魔王＝模試。偏差値でしかトドメを刺せない
7. **小4と中1で味付けを変える**：小4=物語・ペット・収集、中1=ランク・タイムアタック・効率

## 2. ゲームコンセプト

**タイトル案**：まなびの王国クエスト
**あらすじ**：まなびの王国から「ちえのかけら」が魔王シグマ（偏差値の壁の化身）に奪われた。5つの大陸（＝5教科）を旅して仲間（動物先生10体）を集め、かけらを取り戻し、魔王城で偏差値65の壁を打ち破る。
**既存資産の活用**：
- 動物キャラ10体（SVG・音声・セリフ実装済み）→ **仲間（パーティメンバー）**。教科の大陸をクリアすると担当キャラが加入
- トラのカイ（isKai・既存の「難問モード」キャラ）→ **ライバル枠**。各大陸で立ちはだかり、難問で勝負を挑んでくる
- ポイント（c_points）→ そのまま**ゴールド**として表示（経済を一本化し、既存の「ごほうび」設定とも矛盾しない）

---

## 3. ゲーム構造（ワールド → 章 → ノード）

```
ワールドマップ（5大陸 = 5教科）
 └ 章（大陸ごとに3〜6章 = 単元グループ。SUBJECTS/QUESTIONSの単元IDと対応）
    └ ノード（マップ上のマス。1ノード = 1バトル = 5〜8問）
       ├ ザコ戦：単元の練習問題（buildMixLeveled / genQuestion）
       ├ 中ボス：苦手トピック戦（topic_statsから自動選定 = 苦手ドリルのRPG化）
       ├ 復習の森：SRS・まちがいノートの期日分（dueMistakes）を「亡霊」として再戦
       ├ 章ボス：単元まとめテスト10問（HPが高い・弱点記録が次回に持ち越し）
       └ 魔王城（最終）：模試そのもの（buildExam）。魔王のHPは偏差値でしか削れない
```

### 魔王戦 = 模試の接続（偏差値65の可視化）
- 魔王のHPは「必要偏差値」で表示（例：残りHP＝偏差値65の壁）
- 模試を受ける＝魔王への挑戦。**偏差値60で魔王が片膝をつく、65以上でトドメ**という演出
- 敗北してもゲームオーバーにしない：**「修行イベント」**（まちがいノートから5問）でバフを獲得して再挑戦（負け＝復習の動機づけに転換）
- 実模試の校正（calibOffset）済み偏差値を使う＝ゲーム内の勝利が現実の成績と一致する

## 4. コアループ（1日10〜20分）

1. **冒険マップを開く** → 今日のクエスト3択：①メイン進行（新しい単元） ②復習の森（SRS期日分。**期日があるときは森が「呪われて」見え、優先誘導**） ③修行場（苦手単元）
2. **バトル**（下記システム）
3. **リザルト**：XP・ゴールド・ドロップ品（図鑑コレクション）・レベルアップ演出
4. **スタミナ制**：1日3ダンジョン（=約20問。デイリー目標と連動）。使い切ったら「キャンプ」画面（今日の成果・明日の予告）
   - 継続日数でペットが育つ（既存の連続日数・おやすみチケットと整合）

## 5. バトルシステム（学習との対応）

| ゲーム側 | 学習側の実体 |
|---|---|
| 敵に攻撃 | 問題に正解（既存 handleAnswer → onAnswered をフック） |
| ダメージ量 | 難易度（★の数）×初見ボーナス×スピードボーナス（暗算級の速答で会心の一撃） |
| 被弾（自分のHPが減る） | 不正解。**ただし即死せず「弱点スキャン」＝解説表示 → 類題で「リベンジ攻撃」（成功なら半分ダメージ回復）** |
| 敵の強さ | topic_statsの正答率から自動調整（正答率70〜80%帯を狙う適応難易度） |
| 連続不正解 | 仲間キャラが自動でヒント（既存hint）＋敵が弱体化。1バトルで5問つまずいたら保護者LINE通知（line_endpoint） |
| 全滅 | 「修行イベント」（まちがいノート再戦）でバフ→再挑戦。ペナルティなし |

## 6. 成長・報酬系

- **XP／レベル**：正解で獲得（onAnsweredに1行フック）。レベル＝累計学習量の可視化。レベルアップで仲間のセリフ・称号
- **装備・図鑑（コレクション）**：ドロップは**見た目と演出のみ**（例：カブト、マント、武器スキン）。学習を有利にする効果は付けない（報酬設計の落とし穴を回避）
- **ペット**（小4向け）：卵→孵化→進化。餌＝連続日数とデイリー達成。Prodigy研究で愛着が最強の継続要因
- **ランク戦**（中1向け）：週間クエスト（既存QUESTS）と連動した週間ランク（ブロンズ→シルバー→…）。タイムアタックダンジョン
- **ゴールド**：c_pointsをそのまま表示。既存「ごほうび」（親が決めた実報酬）への導線は維持

## 8. 失敗パターン対策チェックリスト（リリース前に確認）

- [ ] 問題を解かずに進行・強化できる要素がない（ガチャ・放置報酬の類を入れない）
- [ ] 不正解時に必ず「解説→類題」が挟まる（答えの丸暗記ループを作らない）
- [ ] 1日の上限（スタミナ）があり、上限後は誘導しない
- [ ] 報酬は見た目・収集・称号のみ。学習ショートカットを売らない
- [ ] つまずき（連続不正解）が大人に見える（LINE通知・記録画面）
- [ ] ゲームをやらない選択でも既存機能（練習・模試）が今まで通り使える

## 9. 技術構成（既存コードへの組み込み）

### 9-1. 挿入点（構造調査に基づく）
- **画面**：`<div id="rpg-screen" style="display:none">` を新設し、`hideMainScreens()` の配列（index.html L3985付近）に `'rpg-screen'` を追記。トップナビ（L929〜）に `🗡️ 冒険` ボタン追加。`renderDailyMenu` のクイックボタン群にも入口を追加
- **状態**：`MU_PER_USER`（L1948）に `rpg_state:1` を追加 → `u:<uid>:rpg_state` としてユーザー別保存・クラウド同期・完全リセットに自動対応
- **バトル**：既存 `runQuestionSession(qs, title, intro, char)`（L5010）を包む `startRpgBattle(node)` を新設。正誤フックは `onAnswered(q, correct)`（L4819）にRPG分岐を1箇所追加（XP・ダメージ計算）
- **問題ソース**：ザコ戦= `buildMixLeveled(area,count,level)` ／復習の森= `dueMistakes()` ／中ボス= `weakTopics()` ベース／魔王= `startExam(area)` をRPG演出でラップ
- **演出**：`CHARS`のSVG・`showToast`・`confetti`・ElevenLabs音声を流用。新規に必要なのは敵SVG数体とHPバーCSSのみ

### 9-2. データモデル（rpg_state・JSON 1キー）
```json
{
  "v": 1,
  "level": 3, "xp": 120,
  "chapter": {"math": 2, "japanese": 1},
  "cleared": {"math:2:node5": 1},
  "collection": {"helm_bronze": 1, "pet_egg": 1},
  "pet": {"stage": 1, "fed": "2026-07-05"},
  "stamina": {"date": "2026-07-05", "used": 1},
  "boss": {"math:2": {"tries": 1, "weak": ["decimal"]}}
}
```
- クラウド同期の注意：既定マージは「クラウド優先」。`cleared`/`collection` はunion、`xp`/`level` はmaxでマージするのが理想なので、実装時に cloud-sync.js の `mergeKey` に `:rpg_state$` 専用マージを追加する

### 9-3. コンテンツ定義（静的配列）
```js
const RPG_CHAPTERS = {
  math: [
    { id:'m1', title:'計算の平原', units:['decimal','fraction'], nodes:8, boss:'スライム王' },
    { id:'m2', title:'図形の森',   units:['plane','solid'],      nodes:8, boss:'アングラー' },
    ...
  ], ...
};
```
単元ID（`SUBJECTS`/`QUESTIONS`のキー）と対応させるだけで、出題・苦手統計・SRSがすべて既存機構に乗る。
