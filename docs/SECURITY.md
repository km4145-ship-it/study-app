## 🔴 未対応（実害あり・外部販売前に必ず塞ぐ）

### 1. Firestoreルールが家族コードを検証していない
- **内容**：ルール（`firestore.rules`、2026-07-14にコード化）は `request.auth != null` のみを要求する。
  匿名認証さえ通れば、4桁の家族コード（既定 `0000`）を知っている／推測できる誰でも
  `families/{コード}` 配下を読み書きできる。メンバーシップの検証が一切無い。
- **影響**：学習履歴・氏名・偏差値等の閲覧、データの書き換え・削除が第三者から可能。
- **対策**：`docs/APP_STORE_ROADMAP.md` の Phase 4（アカウントモデルへの移行）で構造的に解決。
  それまでは「公開URLを他人に教える場合は家族コードを `0000` から変更する」运用でしのぐ
  （`はじめにお読みください_公開と同期の手順.md` に記載済みの既存の注意書き）。
- **発見日**：2026-07-14

### 2. ElevenLabs APIキーが平文でクラウド同期・クライアントから直接使用されている
- **ステータス：🟡 Workersデプロイ・Firestore疎通確認済み（2026-07-14）。実キーでの動作確認＋
  クライアント切替コミットのpush待ち。**
  残作業の手順は `docs/APP_STORE_ROADMAP.md` Phase 2「デプロイ実績」参照。
  **pushが完了するまでは本項の穴は現状のまま残っている**（サーバーを立てただけでは効果が無い。
  クライアントが新URLを使うよう切り替わって初めて閉じる）。
- **内容**：`el_api_key` が `cloud-sync.js` の `SHARED` 配列に入っており、
  `families/{コード}/shared/settings` に平文で同期される。さらに `index.html` の3箇所
  （キー検証・ボイス追加・音声合成）で `fetch('https://api.elevenlabs.io/...', {headers:{'xi-api-key':elevenLabsKey}})`
  と**ブラウザから鍵を直接使ってAPIを叩いている**。
- **影響**：家族コードを知る者は上記1の穴経由でこの鍵を読み取れ、ElevenLabs側の従量課金を
  第三者に使われうる（実害＝課金被害）。
- **暫定対策（ユーザー側で今すぐ実施を推奨）**：ElevenLabsダッシュボードでこのAPIキーの
  利用上限／スコープを制限する。私（Claude）からはアカウントアクセス権が無いため実行不可。
  **デプロイが完了するまではこの暫定対策が唯一の防御**。
- **恒久対策（実装済み・デプロイ待ち）**：`workers/tts-proxy/`（Cloudflare Workers）でTTSを
  サーバー側プロキシ化。鍵はFirestoreの`families/{code}/private/tts`にのみ保存し、GCPサービス
  アカウント経由（Firestoreセキュリティルールを迂回するAdmin相当の特権）でしか読み書きできない。
  さらに1家族あたり1日50,000文字の日次クォータで、家族コード推測経由の乱用被害にも上限を
  設けた（Phase 4のアカウントモデルまでは「鍵は守れるが完全な乱用防止ではない」ことを
  正直に記録しておく）。**当初はFirebase Cloud Functionsで実装したが、Blazeプラン
  （従量課金）が必須という制約があり、課金リスクを避けるためCloudflare Workers
  （無料枠は上限到達時に課金ではなくリクエスト拒否＝構造的に青天井になりえない）へ
  設計変更した（2026-07-15）。詳細は `docs/APP_STORE_ROADMAP.md` Phase 2参照。**
- **発見日**：2026-07-14

## 🟡 現状把握・許容範囲（将来的な見直し候補）

### 3. 顔・指紋認証（WebAuthn）と管理者PINの scope
- WebAuthnの資格情報は**ユーザーごと**に `mu_users` 配列内の `bioCreds[]` に保存される
  （`mu_users` 自体は `SHARED` にあるため同期される＝「この人はbio登録済み」という事実は全端末に伝わるが、
  資格情報自体はデバイス外に出せないプラットフォーム制約により端末ローカルでのみ機能する）。
- 管理者PIN（`mu_admin_pin`、`_pinObf`で難読化）は**家族全体で共有の単一シークレット**。
  設定画面のガード（`muRequireAdmin()`）とbioの代替手段を兼ねる。
- 現状は家族内利用が前提なので許容。Phase 5で親アカウントの実ログインが入ったあとも、
  子どものプロフィール切替UXとしてはこのPIN/WebAuthnの仕組みをそのまま流用する計画。

### 4. Firebase apiKey が公開されている
- `cloud-sync.js` に書かれている `firebaseConfig.apiKey` は Firebase の設計上公開して良いもの
  （保護すべきはルール側）。`はじめにお読みください_公開と同期の手順.md` にも明記済み。**対応不要**。

### 5. 依存関係・サードパーティ素材のライセンス
- 絵文字（装備・モンスター等をcanvasに焼く方式）はOS標準絵文字を使用しており、
  商用配布時のライセンス上グレー。Twemoji（CC-BY）等への差し替えを Phase 3〜7のどこかで検討要。
- BGM（`assets/bgm/*.m4a`）の出所・利用許諾を確認し記録すること（未着手）。
- 問題コンテンツ（`js/generators*.js`）は手続き生成または自作の検証済み事実テーブルのみを
  使用しており、著作権的にクリーン。**これは強み・変更不要**。
