# 物語構成設計 — 「まなびの王国」大陸アーク（10x コンテンツ計画）

タクトの物語を今の約10倍（ステージ13→~150、物語~20行→~250場面、ボス6→~55、
キャスト8→15+）に拡張する設計。決定事項：**1章＝3ノード（濃い構造）／新キャラも追加／
「数の大陸」を縦に完成→型化して他4大陸を量産**。

本書は (A) 全体の枠、(B) 縦スライス＝**数の大陸の詳細設計図**、(C) 量産テンプレ、
(D) 技術アーキテクチャ、(E) フェーズ計画。数の大陸の型が固まれば、他4大陸は
「章データ＋シーン文」を差し替えるだけで量産できる。

---

## A. 世界の枠（通し糸）

まなびの王国が魔王シグマの「モヤの霧」に沈み、ちえがモンスターに変わり、5人の動物先生が
霧に囚われた。見習い勇者（プレイヤー）が5大陸を巡り、単元を攻略しながら先生を救い、
5つの「ちえのクリスタル」を集め、魔王城で決戦する。

```
プロローグ（王国崩壊・使命・シグマ登場）
 ├ 数の大陸アーク   （🔴 かずのクリスタル・コタロウ先生）
 ├ ことばの大陸アーク（🟣 ことばのクリスタル・ミケ先生）
 ├ アルファベット大陸（🟢 えいごのクリスタル・ラビィ先生）
 ├ じっけんの大陸    （🔵 かがくのクリスタル・ナナ博士）
 ├ れきしの大陸      （🟡 れきしのクリスタル・クマ先生）
 ├ 魔王城アーク（5先生と合流・シグマ決戦・過去の真実）
 └ エンディング → エンドゲーム（挑戦の塔・日替わり・裏ボス「??」）
```

各大陸アーク＝**10章**（その教科の単元カリキュラム Lv1→10）。1章＝**3ノード**
（雑魚2＋章ボス1）。**単元＝敵の弱点教科**にして、学習（教科えらび攻撃）と物語を一致させる。

物語の型（10章の起承転結）：
- **起**（ch1）：大陸に到着、被害を知る、コタロウらの導入
- **承**（ch2-4）：単元を1つずつ攻略、日常と手がかり
- **転**（ch5）：シグマ幹部が立ちはだかる／シグマの問答（教科の意義を試す）
- **承**（ch6-9）：単元が深く難しく、勇者の成長、ライバルとの再会
- **結**（ch10）：大陸ボス撃破→先生救出→クリスタル獲得→次への引き

## B. 縦スライス：数の大陸（プロトタイプ＝他4大陸のテンプレ）

大陸テーマ：**計算の力**（お店・時計・くらしを支える力）。弱点教科＝算数(math)。
先生＝コタロウ（shiba）。クリスタル＝🔴かずのクリスタル。

### 新キャスト（数の大陸で初登場、以降の大陸に再登場）
- **レン（ライバル見習い勇者・cat系の別色 or 新art）**：ch3初登場。腕自慢で先を急ぐが根は仲間思い。
  各大陸で再会し、ch7前後で共闘。プレイヤーを鼓舞する存在。
- **天秤の魔神ゼロン（シグマ幹部・数担当）**：ch5の壁。「計算など無意味」と問答を挑む中ボス。
  各大陸に対応する幹部を配置（5幹部）。魔王城アークで再登場。
- **相棒モーグ（小さな精霊・新art）**：プロローグ後半で勇者に同行。行動ヒント役・コミカル。
- 既存：ミミ学園長(owl)／コタロウ(shiba)／シグマ(villain)／たぬすけ行商人(merchant)／巻物ナレーション。

### 物語シーン（配信ポイント）
1. **章導入シーン**：各章の最初のノードに入る前（`{who,char,text}` のタップ送り）。ch1/5/10は手厚く（起承転結の要）、ch2-4/6-9は2-3行。
2. **章ボス撃破シーン**：短い勝利ダイアログ（1-2行）。
3. **大陸クリアシーン**（ch10撃破）：コタロウ救出→クリスタル授与→次大陸への引き（既存クリスタル演出と接続）。
4. **戦闘中ヒント**：既存の先生実況（弱点教科の助言）を章の単元に合わせて。

## C. 量産テンプレ（他4大陸）

各大陸は `RPG_PLAN.<area>`（既に10章分のtopic/boss/node/mons完備）から生成。差分は：
- **教科モンスター**：ことば＝カンジオニ/インクブロブ/フデバード、英語＝グラマロ/ABキューブ、
  理科＝フラスクン/ボルトドレイク、社会＝ハニワ/トキオウ（`js/srpg-mons.js`に既存）を敵テンプレ化。
- **幹部＆先生救出＆アーク文**：`RPG_STORY`の`<area>_ch1/5/10`（手書き済み）を移植＋章2-4/6-9を執筆。
- **弱点教科**＝その大陸の教科。

## D. 技術アーキテクチャ（データ駆動＝10xの肝）

1. **章データ** `SRPG_CONTINENTS`（js/srpg.js or 新ファイル）：`RPG_PLAN`を基に、各章に
   ノード3件（敵配置・地形・ボス・単元）を定義。全部データ。
2. **ステージ生成** `srpgChapterStage(area, ch, node)`：章データから`SRPG_STAGES`相当の
   ステージを動的生成（敵テンプレ・レベル・弱点・地形・boss旗）。
3. **物語データ** `js/story-data.js`：シーンを`{who,char,text}` で外出し（`RPG_STORY`移植＋拡張）。
   キーは `area_ch<n>_intro` / `area_ch<n>_win` / `area_clear` など。
4. **シーンengine**：`srpgStoryIntro`（タップ送り・読み上げ）を多キャラ立ち絵対応に拡張。
5. **敵テンプレ**：教科ボス/モンスターを`SRPG_ENEMY_TEMPLATES`に追加（art＝`SRPG_MON_ART`、弱点＝教科）。
6. **ステージ選択UI**：大陸→章→ノードの階層ナビ（進行で順次解放）。現行の平置きquestを置換。
7. **進行/クリスタル**：ch10ボス撃破で大陸クリア＝クリスタル獲得（既存`srpg_cleared`/クリスタル系に接続）。
8. **純粋関数＋テスト**：生成器・進行判定は純粋関数化し`unit-srpg`で検証（全ノードの整合・敵実在・盤内）。

## E. フェーズ計画

- **P1 基盤＋数の大陸ch1（縦の証明）**：章データ構造・`srpgChapterStage`生成・story-data・
  シーンengine・敵テンプレ（スラッグ王）・階層ナビ。ch1が物語つきで遊べる状態に。
- **P2 数の大陸 完成**：ch2-10のデータ＋シーン、幹部ゼロン(ch5)、ライバル レン、相棒モーグ、
  救出＋クリスタル。数の大陸30ステージが通しで遊べる＝**型の確定**。
- **P3 量産**：ことば/英語/理科/社会を型に流し込み（各10章）。教科モンスター敵テンプレ化。
- **P4 魔王城アーク＋エンディング＋エンドゲーム物語**：5先生合流・シグマの過去・裏ボス。
- 各フェーズ末に build/globals/全テスト/e2e で検証。プレイ可能性を最優先。
