# 学習アカデミー（千咲）— 公開＆同期セットアップ手順

このフォルダ `chisaki-app` は、そのまま Web に公開できる「アプリ一式」です。
Mac・iPad・iPhone で、ホーム画面に追加して“アプリのように”使え、更新も自動で届きます。

## フォルダの中身
- **index.html** … アプリ本体（これを公開します）
- **manifest.webmanifest / sw.js / icons/** … PWA用（ホーム画面追加・オフライン・自動更新）
- **cloud-sync.js** … 任意のクラウド同期（Firebase）。設定するまでは**オフ**（今まで通り端末内保存）

---

## A. GitHub Pages で公開する（無料・プッシュ更新つき）
1. **GitHubアカウント**を作る（無料） → https://github.com
2. **New repository**（新規リポジトリ）を作成。名前は例 `chisaki-app`、**Public** を選択。
3. このフォルダの中身を**全部アップロード**（`index.html` / `manifest.webmanifest` / `sw.js` / `cloud-sync.js` と **icons フォルダ**）。
   - ブラウザだけでOK：リポジトリ画面の **Add file → Upload files** にドラッグ&ドロップ → **Commit**。
4. リポジトリの **Settings → Pages** を開き、**Branch: main / (root)** を選んで **Save**。
5. 数分で公開URLが出ます：`https://ユーザー名.github.io/chisaki-app/`
6. 家族はこのURLを開くだけ。

### 更新のしかた（プッシュアップグレード）
1. アプリを直したら、新しい **index.html** を同じ場所に**アップロードして差し替え**。
2. **sw.js の `VERSION` を1つ上げる**（例 `v1.0.0` → `v1.0.1`）。
3. 家族の端末では、次に開いたとき **「✨ 新しいバージョンがあります → 更新」** が出て最新になります。

---

## B. Mac / iPad / iPhone で“アプリ化”（ホーム画面に追加）
- **iPhone / iPad（Safari）**：公開URLを開く → 共有ボタン → **「ホーム画面に追加」**。アイコンから全画面で起動、オフラインでも動作。
- **Mac（Safari）**：URLを開く → 共有 → **「Dockに追加」**（Chrome ならアドレスバーのインストールアイコン）。
- 一度開けば、オフラインでも使えます。

---

## C. クラウド同期（Firebase）— どの端末でも「続きから」
設定済みのFirebaseに、**ユーザーごとに1ドキュメント**で保存されます（v2）。
- 保存先：`families/{家族コード}/members/{ユーザーID}`（学習データ）と `families/{家族コード}/shared/settings`（共通設定）
- 家族コードは既定で `0000`。変えたいときは **⚙️設定 → 「☁️ クラウド同期（家族コード）」**（全端末で同じコードにする）。
- ユーザー選択画面で名前をタップすると、**クラウドの最新データを読み込んでから**始まります。
- 旧形式（v1：家族全体で1ドキュメント）のデータは、初回起動時に自動で新形式へ移行されます。

### 顔・指紋認証（Face ID / Touch ID / Windows Hello）
- **⚙️設定 → ユーザー → 「🔓認証なし」ボタン**を押すと、その端末の顔／指紋を登録して🔒ロックできます。
- ロック中のユーザーは、選択時に顔／指紋の確認が入ります。使えない端末では**管理者の合言葉(PIN)**で開けられ、そのとき「この端末でも登録する？」と聞かれます。
- 利用条件：https での公開（GitHub Pages はOK）＋生体認証を設定済みの端末。

### Firestore セキュリティルール（コンソール → Firestore → ルール）
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /families/{fam} {
      allow read, write: if request.auth != null;
      match /{document=**} {
        allow read, write: if request.auth != null;
      }
    }
  }
}
```
- firebaseConfig の apiKey は**公開されてよい**設計です（守るのはルール側）。秘密鍵は置かないこと。
- 家族コードを知っている人だけがデータにたどり着けます。公開URLを他人に教える場合は、既定の `0000` から**必ず変更**してください。
- 無料枠：保存1GiB・読み5万/日・書き2万/日（家族用には十分）。

## メモ
- **ローカルでファイルを直接開いても、今まで通り動きます。** ただし「ホーム画面追加・自動更新・クラウド同期」は **URLで公開してから**有効になります（ブラウザの仕様）。
- 今後アプリを改良したら、**index.html を更新 → 再アップロード → sw.js の VERSION を上げる**、の流れです。
- 子どものデータなので、リポジトリには成績以外の個人情報を入れない運用がおすすめです。

## D. 合言葉ロック（公開URLを家族以外に開かれたくないとき・任意）
公開URLは誰でも開けますが、アプリ側に「合言葉（PIN）」を設定すると、起動時に入力を求められます（簡易的な目隠し）。
1. アプリの **⚙️設定 → 「🔒 合言葉ロック」 → 「合言葉を設定／変更」** で合言葉を決める。
2. 次に開くときから、合言葉を入れないと中に入れません。
3. 自分の端末では「この端末では次回から省略」にチェックすると、毎回の入力を省けます（知らない人の端末では合言葉が必要なまま）。
4. やめたいときは同じ画面の「解除」。
※ これは“簡易的な目隠し”です（中身を強く守る暗号ではありません）。本当に大切なデータは Firestore のルールと PIN で守られます。

## E. 2026-07 アップデート（v1.1.0）で増えた機能

### 💬 LINE通知（保護者向け）
学習結果（模試の偏差値・今日の目標達成・毎晩のまとめ・未学習リマインド）を保護者のLINEに届けられます。
設定方法は **「LINE通知の設定手順.md」** を見てください（無料・約20分）。
設定後、アプリの **⚙️設定 → 「💬 LINE通知」** にURLを貼るだけです。

### 📈 偏差値の校正（精度アップ）
- 学校や塾の**本物の模試の偏差値**を **⚙️設定 → 「📈 偏差値の校正」** から入力すると、アプリの偏差値がその結果に合うよう自動補正されます。**模試のたびに入力するのがおすすめ**です。
- 模試は「第N回」として問題構成が固定され（基礎9問・標準10問・応用6問）、回による難易度のブレが減りました。
- 選択問題の「当て推量」も補正して計算します。1回の結果より **「直近3回平均（±幅）」** を実力の目安にしてください。

### 🖨️ 印刷の強化＆紙の答え合わせ
- 「🖨️ プリント」から **種類（練習／模擬試験／まちがいノート）・教科・レベル・問題数** を選んで印刷できます。図形問題の図も印刷されます。
- 印刷すると自動で「控え」が残り、解いたあと **「📥 紙の答え合わせ」** で⭕❌をつけると、学習記録・苦手分析・まちがいノート・今日の目標に反映されます。**紙の模擬試験は偏差値も記録されます。**

### ⭐ 続けやすくする仕組み
- **週間クエスト**：今週80問・週5日学習・模試1回・復習10問前進、達成でボーナスポイント。
- **おやすみチケット**：週1回までなら1日休んでも連続日数がリセットされません。
- **復習の間隔**が 1→3→7→14→30日 に延長。1か月後の再確認に合格してから「卒業」になります。

### ➕ 問題の追加
**⚙️設定 → 「➕ 問題の追加読み込み」** からJSONファイルで問題を追加できます。形式は **「追加問題サンプル.json」** を参照。ChatGPTやClaudeに「このJSON形式で中1数学の方程式の問題を20問作って」と頼むと量産できます。

### ⚠️ iPhone/iPadの注意（データ消失対策）
SafariでURLを開いただけの状態だと、**7日間使わないとブラウザがデータを消す**ことがあります。必ず **「ホーム画面に追加」** して使い、できればクラウド同期（上のC）も設定してください。
