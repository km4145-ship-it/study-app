# 学習アプリ改善レビュー（目標：偏差値65以上）

レビュー日：2026-07-02　対象：index.html（4490行）／cloud-sync.js／sw.js

## 現状の全体評価

単一HTMLのPWAとして完成度は高い。5教科の練習・模試（25問50分・配点つき）・偏差値換算・A〜E判定・偏差値65ロードマップ・まちがいノート＋間隔反復（1/3/7日）・苦手分析・日課/連続日数/カレンダー・バッジ16種・ごほうび・キャラ＋音声・印刷・Firebase同期（未設定）・オフライン対応まで揃っている。

ただし「偏差値65」という目標に対しては、**偏差値の測定精度・複数端末同期・LINE通知・印刷の実用性**に大きな穴がある。

---

## A.【最重要】偏差値の正確性

現状の計算：`偏差値 = 50 + 10 ×（得点率 − 想定平均）/ 想定標準偏差`（index.html:3716-3732）。想定平均・標準偏差は教科ごとの**固定の推定値**（例：数学 mu=0.57, sigma=0.19）。

1. **実在の母集団に基づいていない** — アプリ内の得点率を固定係数で変換しているだけ。本物の模試とのズレを補正する仕組みがない。
   → 改善：学校・塾の実模試結果（点数・偏差値）を手入力する画面を作り、アプリ偏差値との差分で mu/sigma を自動校正する。
2. **出題が毎回ランダム生成混在**（buildExam）— 回によって難易度が揺れ、偏差値が実力でなく運で±数ポイント変動する。
   → 改善：等化した固定フォーム（第1回・第2回…）を用意し、同一フォームでの再受験を防ぐ。
3. **練習と模試の問題プールが同じ**（MATH_EXAMが苦手ドリル・印刷にも流用）— 問題を覚えてしまい偏差値が上振れする。
   → 改善：模試専用プールを分離し、練習に出た問題は模試から除外。
4. **25問1回の測定誤差は±4〜5** — 小数第1位表示（例 63.4）は過剰精度で、1回の結果に一喜一憂させる。
   → 改善：直近3回の移動平均を「現在地」とし、「62±3」のような幅つき表示に。
5. **4択の当て推量補正なし** — 全部勘でも期待25%得点。低学力帯の偏差値が上振れ。
6. **範囲が「中1・8月まで」固定** — 時間が経つと実際の試験範囲と乖離する。テスト日設定と連動した範囲拡張が必要。
7. **判定の不整合** — A判定が偏差値66以上（judgeOf）なのに目標・バッジは65。65に統一すべき。
8. **全角数字の入力が不正解になる**（checkFreeAnswer index.html:2403）— norm()が＋−と読点しか正規化せず、iPhone/iPadの日本語キーボードで「１２」と入れると誤答扱い。**偏差値が下振れし、理不尽な×でやる気も削ぐ**。全角→半角変換・英字の大文字小文字・×/*・スペース類の正規化を追加。

## B. 複数端末（PC・iPad・iPhone）

1. **Firebase同期が未設定**（cloud-sync.js の config が PASTE のまま）— 現状は端末ごとに記録がバラバラ。最優先で設定する。
2. **同期マージに欠陥** — 配列・カウンタ以外のキーは「ローカル優先」（applyCloud）。`topic_stats`（苦手分析の本体）や `daily_prog` はオブジェクトなので、端末間で分岐したまま**片方の学習履歴が永久に反映されない**。
   → 改善：topic_stats は attempts/correct をキー毎に合算マージ、その他はキー毎タイムスタンプの新しい方優先に。
3. **iOSのデータ消失リスク** — Safariは7日間未使用のサイトのlocalStorageを削除することがある。ホーム画面追加を必須の運用にし、クラウド同期＋自動バックアップで防御。
4. member ID が「名前_PIN」の平文 — 家族用途なら許容だが、Firestoreセキュリティルールの設定は必須（READMEに手順を明記）。

## C. LINE通知（未実装）

- コード内に通知機能は一切ない。
- **LINE Notifyは2025年3月でサービス終了済み**のため、LINE Messaging API を使う。
- 構成案（無料で可能）：アプリ→Firestoreに学習イベント保存→**Google Apps Script（時間トリガー）** が毎晩、保護者のLINEへダイジェスト送信。
  - 通知内容：今日の問題数・正答率・偏差値の更新・連続日数・「今日まだやっていません」リマインド（20時）
- 補助：PWAのWebプッシュ（iOS 16.4以降・ホーム画面追加時のみ）で子供本人への学習リマインド。

## D. 印刷（現状は最小限）

printSheet（index.html:3552）は**全問題からランダム10問**を出すだけ。

1. 教科・単元・レベル・範囲の選択ができない（小学生問題と中1問題が混在して出る）
2. **図形問題のSVGが印刷に含まれない** — 図なしで問題文だけ印刷される
3. **模試の紙印刷がない** — 本番形式（50分・配点・解答用紙つき）を紙でやれるべき
4. **まちがいノートの印刷がない** — 紙での解き直しは定着に最も効く
5. **紙で解いた結果をアプリに戻せない** — 「紙の答え合わせ入力」画面を作り、学習記録・苦手分析・SRSに反映させる（これがないと紙学習が記録から抜け落ちる）

## E. 飽きさせない仕組み・継続性

良い点：キャラ10体＋音声、バッジ、連続日数、日課リング、カレンダー、ごほうび、テストカウントダウン。

1. **問題が枯渇する** — 静的問題472問＋浅い生成パターン。毎日20問なら数週間でパターンを覚える。問題データをJSONに外部化し、追加・差し替えを容易に（Claude等で単元別に量産する運用）。
2. **連続日数が1日欠けで0リセット** — 継続系で最も離脱を生む仕様。「おやすみチケット」（週1回まで連続維持）を導入。
3. **バッジは取り切ったら終わり** — 週間クエスト（例：今週 苦手単元を3回克服）、レベル/称号、キャラの進捗解放に転用して長期目標を作る。
4. **SRSが3段階（1/3/7日）で卒業後は二度と出ない** — 忘却する。14日・30日を追加し、卒業問題の月次総復習を自動出題。
5. **設定UIがprompt()/alert()** — iPadで使いにくい。モーダルUIに置き換え。

## F. 学習記録

1. study_log は直近500件・まちがいノート200件で切り捨て — 1年続けると古い記録が消える。クラウド側は全量保持に。
2. セッション単位の記録のみで**問題単位のログがない** — 「どの問題に何秒かけて何回間違えたか」がないと精密な苦手分析ができない。
3. バックアップは手動エクスポートのみ — 同期設定後は自動化される想定だが、未設定の現状では消失リスク大。
